在 Java 中，共有三种 IO 模型，常用的有 BIO (Blocking I/O 同步阻塞)、NIO (non-blocking I/O 同步非阻塞)、AIO (Asynchronous I/O 异步非阻塞)。通常常用的只有 BIO、NIO。

# BIO：同步阻塞的 I/O 模型

简介：Java 1.4 以前的唯一选择，存在于 java.net 的包中，它是一种单线程的 I/O 模型，适用于一场连接较少且固定的架构，优点是程序简单，易于理解。面向流编程。

BIO_DEMO:

  • BIO 一个服务端只能接受一个客户端的连接。当没有连接进来时，会处于阻塞的状态。当连接了一个客户端后，其他的客户端就无法再次连接。

如果需要同时连接多个客户端，就需要开启多线程。

  • 如图，每进来一个客户端时，都需要开启一个线程处理新的客户端，一个线程只能处理一个客户端的连接。

    使用线程池

  • 总结：BIO 的编码较为简单，适用于连接数目较少且固定的架构，对于一些高并发下的场景，BIO 无法胜任，因为它的每一次连接都需要一个线程来进行维护，且每个客户端的每个连接不是一直处于连接状态，但连接必须要维护，会造成不必要的资源浪费，严重依赖于线程，会带来 cpu 使用率高等问题。

# NIO (Non Blocking I/O) 同步非阻塞的 I/O 模型

简介：NIO 它也是一个单线程的 I/O 模型，不同于 BIO，它是一个非阻塞的 I/O 模型，是 I/O 多路复用的基础 ，可以解决高并发，多连接的问题。面向缓冲区。

# Buffer（缓冲区）：

与 BIO 不同，BIO 的数据传输依赖于流（InputStream/OutputStream），而 NIO 的数据则是由 Buffer 缓冲区操作。

Buffer 中的核心参数：

  • mark (标记位)：在初始时为 - 1，可以对当前数据的位置进行标记，通过 **reset ()** 方法，将 position 的位置进行重置

  • position (位置)：数据从哪里开始读 / 写，初始值为 0，其值不能大于 limit 界限，可以通过 **postition ()** 进行修改

  • limit (界限)：最多可以读 / 写到哪里，初始值为 capacity 的容量最大值，可以通过 **limit ()** 进行修改

  • capacity (容量)：在 Buffer 初始化时传入，表示一个 Buffer 里面最多能存取多少的数据。

# Channel (通道)：

缓冲区是一个容器，它用来将数据进行存储，而数据进行传递，则就需要依赖于通道。通道是双向的，即可以传递数据，也可以接收数据。但它不能直接访问数据，需要 Buffer 交互，可以进行异步的读写。

Channel 可分为两大类：

  • Socket 套接字 Channel：

    • **DatagramChannel：**UDP 无状态的数据读写通道。
    • **SocketChannel：**TCP 有状态的数据读写的客户端通道
    • **ServerSocketChannel：** 服务端通道，每个客户端连接，都会创建一个 SocketChannel。ServerSocketChannel 只负责监听，其并未实现父类的 read/write 方法

    这三种通道类型，都可以设置中非阻塞模式，通过 configureBlocking (boolean block) 设置为 false 在实例化时，都会创建一个对等的 socket 对象。

# Selectror (选择器，又被称为多路复用器)

selector 选择器，在使用单线程处理多个客户端请求时，就需要使用到 selector。它可以管理注册到选择器中的通道，使用事件驱动的方式，来监听通道中的事件。

  • 总结：NIO 是单线程、同步非阻塞、多路复用的 I/O 模型，引入了 Selecor (选择器，多路复用器)、Channel (通道)、Buffer (缓冲区)，数据存储到 Buffer，通过 Channel 进行传递，Selector 使用了事件驱动的形式，监听所有注册到 Selector 中的 Channel，有事件时，才会进行相应的操作，减少了线程的开销，cpu 切换上下文所消耗的时间，大大提高了效率和减少了系统的资源消耗
